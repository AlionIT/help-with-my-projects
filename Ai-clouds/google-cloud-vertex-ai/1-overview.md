# 🧠 Обзор архитектуры: Библиотека и Авторизация (ADC)

В этом документе описаны два главных архитектурных решения, принятых для работы с Google Cloud Vertex AI.

## 1. Выбор библиотеки: `google-genai`
Для работы с Vertex AI в Python исторически использовалась библиотека `google-cloud-aiplatform`. Однако Google объявил её устаревшей (deprecated) для задач генеративного ИИ. 
**Наш стандарт:** Мы используем новейшую унифицированную библиотеку **`google-genai`**.

## 2. Механизм авторизации: Application Default Credentials (ADC)
Поскольку API Keys запрещены политиками безопасности, мы используем механизм ADC. 

**Что такое ADC:**
Это умный алгоритм поиска доступов, встроенный прямо в библиотеку `google-genai`. При запуске кода (например, `client = genai.Client()`), библиотека начинает искать ключи доступа в системе в определенном порядке. 

Вам **не нужно** хардкодить (вписывать) пароли или пути к ключам внутри Python-скрипта. Скрипт остаётся чистым и безопасным.

**Развилка сред (Local vs Prod):**
Благодаря ADC наш Python-код остается абсолютно идентичным в любой среде. Меняется лишь то, как мы "подкладываем" пропуск в саму операционную систему:

* **Ветка 1: Локальная разработка (Local).** Разработчик использует утилиту `gcloud` на своем ПК. Утилита генерирует временный JSON-токен и прячет его в системную папку (например, `AppData`). ADC находит его там. (См. `2-setup-local.md`).
* **Ветка 2: Продакшн сервер (Prod).** На сервере нет браузера и утилиты `gcloud`. Там мы создаем системного робота (Service Account), скачиваем его ключ в виде `.json` файла и указываем путь к нему через переменную окружения ОС. ADC считывает эту переменную. (См. `3-setup-prod.md`).
